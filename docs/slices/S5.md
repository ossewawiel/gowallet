## 🎯 Prompt (paste into `/design-slice`)

```
/design-slice S5 — CSV batch ingestion

Ingest a CSV of transactions safely — even if the file is reprocessed — with a summary + full audit.

Interface (your choice): POST /batch (multipart CSV upload) OR a CLI entrypoint.
CSV header: ref,account_id,kind,points,occurred_at
Example row: tx-001,member-123,earn,150,2024-06-01T10:00:00Z

Deliver:
- parse + apply each row through the SAME earn/spend service (idempotent on ref)
- a summary once complete: total processed / accepted / rejected / duplicates
- write every row attempt to the audit trail (from S4) with its outcome + reason

Enforce (docs/ACCEPTANCE.md):
- INV-9 reprocessing the same file double-counts nothing (idempotent on ref)
- INV-10 returns the summary counts

Traps:
- Rows for the same account appearing close together must stay correct — reuse S2's atomic
  transaction + single-writer; no special-casing.
- A batch row is just another earn/spend — don't bypass the domain rules.

Deps: S2 (spend) + S4 (audit) merged. Stream: C.
Read: docs/specifications.pdf, docs/REST_API_GUIDELINES.md, docs/ARCHITECTURE.md, docs/ACCEPTANCE.md (INV-9,10).
```
