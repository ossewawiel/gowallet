## 🎯 Prompt (paste into `/design-slice`)

```
/design-slice S4 — Audit trail

A durable record of each transaction attempt (accepted or rejected) with a reason + timestamp.

Deliver:
- an audit table (goose migration, TIMESTAMPED filename) with:
  id, ref, account_id, kind, points, outcome (accepted|rejected|duplicate), reason, created_at
- an audit-writer service (defined in internal/wallet, persisted by internal/sqlitestore) that the
  transaction path and the batch path both call
- GET /audit → ADMIN-ONLY listing of the audit log (each recorded attempt), newest-first; optional
  ?account_id= filter. No pagination (demo scale).

Enforce (docs/ACCEPTANCE.md):
- INV-11 every attempt is audited with a reason + timestamp
- INV-21 GET /audit is admin-only (member → 403) and returns each attempt's outcome + reason + timestamp

Traps:
- Audit is append-only and must never change the correctness of the main transaction.
- Build it standalone now (Stream C) so batch (S5) can lean on it.

Deps: S0 merged. Consumed by S5. Stream: C.
Read: docs/specifications.pdf, docs/ARCHITECTURE.md, docs/ACCEPTANCE.md (INV-11).
```
